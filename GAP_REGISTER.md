# GAP_REGISTER

Objective: refactor this repo so the product API follows the Frontplane mission
and the project stays generic across repositories and ecosystems.

| ID | Requirement | Mission phrase/source | Closure evidence | Status |
| --- | --- | --- | --- | --- |
| FP-01 | Product API behavior belongs in Frontplane, not the Node gateway. | `../../mission_statement`: "IDL to quickly create a zero trust facade API"; user instruction: "all the APIs should be done in frontplane" | `frontplane.fp` contains `ConsoleReports`, `KeywordConfigs`, and `SzzAnalyses`; `ux/server.mjs` normal mode is static/proxy infrastructure. | Closed |
| FP-02 | Outbound provider REST edges are declared in the IDL. | `../../README.md`: "outbound REST is part of the FP layer, not hidden runtime code" | `frontplane.fp` declares `RepointelFacade.*`, `RepointelAnalyticsProvider.*`, and `SzzAnalysisProvider.*`; generated manifest includes provider `Ncall` steps. | Closed |
| FP-03 | Auth must not be role-word demo tokens. | `../../mission_statement`: "zero trust facade API" and "securely stitches together" | Backing runtime rejects `reader`, `writer`, and `admin` bearer tokens and requires explicit token registry; provider auth tokens are explicit env config. | Closed |
| FP-04 | SZZ must cross a declared API boundary and avoid hidden Repointel writes. | `../../README.md`: "Every upstream REST edge should be declared as a top-level `rest Module.call(...)`" | `SzzAnalysisProvider.*` is declared and used by generated `Ncall`; SZZ enrichment no longer inserts Gerrit raw records into `repointel_records`. | Closed |
| FP-05 | Downstream service catalog must not imply dynamic hidden routing. | `../../mission_statement`: "securely stitches together cloud products and services" | Connection tests validate Repointel rows before the declared `RepointelFacade.health` call; README and `docs/state-ownership.md` document routing ownership. | Closed |
| FP-06 | Local state ownership must be explicit. | `../../mission_statement`: "facade API"; user instruction: refactor toward FP mission | `docs/state-ownership.md` separates facade-owned coordination records from Repointel-owned provider records. | Closed |
| FP-07 | Default behavior must stay generic across ecosystems. | `MISSION.md`: "Stay generic across repositories and ecosystems; resist overfitting." | SZZ defaults use `REPOINTEL_GIT_ROOT`, require `REPOINTEL_GERRIT_URL` for Gerrit enrichment, require explicit selection or `params.all=true`, and generic dictionaries/UI defaults no longer bake in Keystone/OpenDev. | Closed |
| FP-08 | Provider contracts should be inspectable by generated artifacts. | `../../mission_statement`: "IDL"; `../../README.md`: "generated code, manifests, policy checks, telemetry, and audits can inspect it" | Provider calls live in `frontplane.fp`; generated manifest is regenerated from the IDL. Splitting into `providers.fp` is not required for this generator path. | Not required |
| FP-09 | Synthetic telemetry must not pretend to know generated provider status. | `../../README.md`: "telemetry, and audits can inspect it" | Synthetic downstream traces use `status_code = 0` with an explicit unavailable-status message; real health traces still record actual success. | Closed |
| FP-10 | Docker deployment proof must carry the DB and exercise a real repository. | User request: "we need to package this as a docker"; "can you do deployment proof, using a openstack repo like keystone"; "Shouldn't the docker for this carry the db via compose?" | `docker-compose.yml` owns `postgres`; `Dockerfile.repointel` builds the generated downstream facade; `DEPLOYMENT_GATE.md` records the Keystone proof with completed ingestion and metadata/SZZ provider calls. | Closed |
| FP-11 | A fresh clone must be self-contained. | User blocker: "Make docker compose up work from a clean clone." | Repointel generated source is vendored under `generated/repointel/`; compose no longer references `../../generated/repointel` or the parent workspace; CI compose smoke test added. | Closed |
| FP-13 | Browser-held service tokens must be replaced by product authentication. | `../../mission_statement`: "zero trust facade API" and "securely stitches together cloud products and services without having to expose them externally"; user instruction: "Browser | Secure session cookie | Gateway / backend for frontend | Internal scoped credentials" and "Add RBAC" | `ux/server.mjs` requires signed gateway sessions, supports OIDC plus bootstrap admin login, sets HttpOnly SameSite cookies with configurable `Secure`, enforces RBAC, strips browser credentials, and injects internal scoped credentials server-side; `ux/login.html`, `ux/index.html`, and `ux/app.js` remove browser token entry/storage and gate debug diagnostics to administrators. | Closed |
| FP-12 | Production deployment must prove the live facade runs this version. | `loop-md` production deployment gate | Local Docker deployment proof is complete. A non-local production/staging promotion still requires target, secrets, and deploy approval. | Blocked |
